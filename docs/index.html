The flagship loop

Flash a map, then prove it on the bus

The reason the Suite exists. ECU Studio writes the change; SocketSpy watches the CAN bus and confirms it landed — at the right RPM, the right load, the right operating point.

1

Edit the map

Open the ROM in ECU Studio, locate the map (A2L, DAMOS, or an OpenDAMOS axis-fingerprint recipe) and apply your change in the 2D/3D editor.

→
2

Flash the ECU

Checksums are corrected automatically and the ROM is written over MPPS, with read-back verification and git history so any state can be restored.

→
3

Verify live on CAN

SocketSpy decodes the live bus and confirms the new behavior at the targeted operating point — closing the loop between what you wrote and what the engine does.

Beta The end-to-end automated loop is new and being hardened — each step already works on its own today.

OpenDAMOS

One recipe, many firmwares

A free, CC0 DAMOS that relocates ECU maps by AXIS FINGERPRINT instead of hardcoded addresses — so a single recipe works across firmware variants. A converter turns a manufacturer DAMOS (A2L) plus a ROM into an open, shareable recipe.

  • Proven Bosch EDC16 (big-endian inline) — verified across firmware variants.
  • Proven Bosch EDC17 (little-endian inline) — verified across firmware variants.
  • Beta COM_AXIS ECUs (PSA Valeo, Continental SID807) — works 11/12 on Valeo; no cross-firmware test yet.

Where things stand

Feature status board

Honest maturity, three columns. Shipped & Proven means verified and stable. Beta means it works but is unproven. Incoming is on the roadmap.

Shipped · Proven

Verified, shipped, stable.

  • 2D map editor — Bosch Kf maps, percentage scaling
  • A2L / DAMOS parser (Bosch & PSA dialects)
  • DAMOS editor — edit recipes, detect maps, export A2L
  • Checksum correction — EDC16 (CRC-16/ARC) + EDC17/MED17, read-back verified
  • Every save auto-commits to a per-project git; original ROM kept immutable (.orig)
  • Heuristic map finder (no A2L needed)
  • ROM compare — byte diff grouped into map intervals
  • Git versioning of ROM history (libgit2)
  • Hex view with map-region overlays
  • In-app auto-update (Ed25519-signed, SHA-256)
  • OpenDAMOS Bosch EDC16 & EDC17 (axis fingerprint)
  • SocketSpy live monitor, DBC decode, signal graphs, transmit
  • Protocol decoders: CANopen / J1939 / ISO-TP / UDS / OBD-II / NMEA2000
  • Lua scripting, CAN simulator, MCP/JSON-RPC API
  • io_uring capture, frame fuzzer, BLF/MDF4 export, capture diff, multi-bus
  • UDS tester + ECU simulator, Signal Detective, i18n

Beta

Works, but unproven.

  • OpenDAMOS COM_AXIS — PSA Valeo & Continental SID807 (11/12 on Valeo)
  • The hub launcher — open & share config across sub-programs
  • Flash → verify loop (ECU Studio ↔ SocketSpy)
  • 3D map view — pseudo-3D surface + heatmap with ghost baseline
  • OBD-II datalog (ELM327) — live PIDs, DTC read & clear, VIN, CAN sniff over a USB ELM327; not yet vehicle-tested
  • AutoMods — one-click Stage 1 / IMMO OFF / depollution recipes; safety-guarded but not field-proven
  • Brand-new SocketSpy panels still being hardened

Incoming

On the roadmap.

  • MPPS read/write ECU ROM over USB — DRAFT: simulation mode only, not yet verified on real hardware
  • Full hub interconnection between all sub-programs
  • More ECUs covered by OpenDAMOS recipes
  • Cross-firmware proof for COM_AXIS ECUs
  • EDC16CP33 complete coverage
  • Native OpenGL 3D map rendering (Q3DSurface)
  • 60fps QPainter virtual-scrolling hex view
